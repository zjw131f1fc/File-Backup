# Frontend

本目录是 File Backup 的静态前端，不依赖 Node 构建工具或第三方运行时依赖。

## 本地运行

在仓库根目录执行：

```bash
python3 -m http.server 4173 --directory frontend
```

然后打开：

```text
http://127.0.0.1:4173/
```

## 当前功能

- 任务总览、状态统计和活动记录
- 备份/还原任务创建表单
- 备份输出目录和可选归档名称，未命名归档自动递增避免重名
- 六类筛选条件输入
- 冲突策略选择
- 本机路径浏览弹窗
- 任务详情、进度和取消操作
- 多任务同时创建、列表展示和状态刷新
- 后端不可用时的演示数据模式

## API 联调

前端按根路径 `/api` 请求 `web_api.md` 中定义的接口。后端不可用时，页面会自动切换到演示模式；后端启动后点击右上角刷新按钮即可重新检查连接。

前后端分端口联调时，使用查询参数指定 API 地址：

```text
http://127.0.0.1:4173/?api=http://127.0.0.1:8080
```

也可以在页面加载前设置 `window.BACKUP_API_BASE`。后端启动时将允许来源配置为 `--origin http://127.0.0.1:4173`。

前端不限制同时创建的任务数量。任务列表会显示所有任务，活动任务统计会同时计算 `PENDING` 和 `RUNNING`，轮询会一次刷新整个任务列表。单个任务详情通过任务 ID 打开，取消操作只影响当前任务。

生产部署时应让前端和 Web API 使用同源地址。开发环境如果前后端分端口运行，需要在后端配置允许的前端 Origin。

## Code map

- `app.js`: 页面状态、任务提交、任务列表、恢复点、路径选择器和事件绑定。
- `app-data.js`: 演示数据、API 请求和纯格式化函数。
- `index.html`: 页面结构和表单字段。
- `styles.css`: 页面布局和视觉样式。
